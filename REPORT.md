# 少儿编程课程学习行为数据处理与分析 — 项目报告

## 一、项目背景

随着素质教育政策推动，少儿编程培训市场快速扩张。本项目基于模拟的少儿编程课程学习行为数据，使用 Python 完成完整的数据分析流程，包括 Mock 数据生成、数据质量检查、数据清洗、特征构造、探索性分析和数据可视化。

**本课程项目信息：**
- 课程名称：数据处理综合训练
- 学院：信息工程学院
- 专业班级：计算机2305
- 起止时间：2026年6月15日至2026年7月3日

## 二、数据来源说明

本项目使用的数据为 AI 辅助生成的 Mock 数据，并非真实爬虫数据。根据课程要求，采用多文件数据设计，共生成 5 个互相关联的 CSV 文件，关系如下：

```
students.csv ──┐
               ├── learning_records.csv ──┐
courses.csv ───┘                          ├── assignments.csv
                                          └── attendance.csv
```

### 2.1 学生信息表 (students.csv)
- 记录数：201 条（含1条重复）
- 字段：student_id, name, age, gender, grade, enrollment_date, course_level, parent_phone

### 2.2 课程信息表 (courses.csv)
- 记录数：20 条
- 字段：course_id, course_name, category, difficulty, duration_weeks, lessons_count

### 2.3 学习记录表 (learning_records.csv)
- 记录数：1205 条（含5条重复）
- 字段：record_id, student_id, course_id, lesson_id, lesson_date, duration_minutes, completion_status, score, code_lines, error_count

### 2.4 作业提交表 (assignments.csv)
- 记录数：800 条
- 字段：assignment_id, student_id, course_id, assignment_type, submit_date, score, time_spent_minutes, attempt_count, submission_status

### 2.5 出勤记录表 (attendance.csv)
- 记录数：1000 条
- 字段：attendance_id, student_id, course_id, class_date, is_present, late_minutes

## 三、数据质量问题

人为注入的数据质量问题包括：
1. **缺失值**：学生表 age/gender/grade/course_level，学习记录的 error_count/score
2. **重复值**：学生表1条重复，学习记录表5条重复
3. **异常值**：年龄3岁、负分-10、分数>100、代码行数-5
4. **格式不统一**：日期格式(Y-m-d vs Y/m/d vs m/d/Y)、年级含文字
5. **业务逻辑异常**：低龄学生选修高级课程

## 四、分析流程

### 4.1 数据加载与查看
使用 pandas 读取全部5个CSV文件，查看基本信息、字段类型和描述性统计。

### 4.2 数据清洗
- 学生表：去重1条，填充缺失值(age中位数填充, gender/level众数填充, grade按年龄推算)，处理年龄异常(修正2条)，统一grade格式，统一日期格式，修正业务逻辑异常(7条低龄-高级课程调整为中级)
- 学习记录：去重5条，填充error_count缺失值(中位数2)，处理分数异常(14条>100修正为100)，修正代码行数负数(13条)，修正异常学习时长(16条>240分钟)
- 作业提交：修正10条负分
- 出勤记录：填充is_present缺失值(5条)，修正迟到时间异常(8条)

### 4.3 特征构造
构造了6个新分析字段：
1. **age_group**: 年龄段分组(低龄/中龄/学龄/少年/青少年)
2. **lesson_weekday/is_weekend/weekday_name**: 学习时间特征
3. **duration_category**: 学习时长类别
4. **is_programming**: 是否为编程类作业
5. **score_level**: 成绩等级划分
6. **category_broad**: 课程大类汇总

### 4.4 探索性分析
围绕7个分析问题进行统计：
1. 不同年龄段学生的课程选择偏好
2. 不同年龄段的学习完成率和平均得分
3. 一周中各天的学习活跃度
4. 不同课程类别的学习效果对比
5. 学习时长与得分的相关性（Pearson相关系数: 0.0473）
6. 作业提交情况分析
7. 出勤率分析（总体出勤率: 84.1%）

## 六、主要结论

### 6.1 年龄与课程偏好
低龄学生（7-8岁）主要选择图形化编程课程，随着年龄增长，代码编程类课程的占比逐渐上升，青少年阶段（15-16岁）更倾向于代码编程和竞赛类课程。

### 6.2 学习完成率
中龄（9-10岁）和学龄（11-12岁）学生的课程完成率最高，低龄学生因注意力集中时间较短，完成率相对较低，需在课程设计上增加互动性和趣味性。

### 6.3 学习时间规律
学生学习活动高度集中在周末（周六、周日），周中学习次数明显偏少，这符合少儿编程作为课外辅导的定位。

### 6.4 学习时长与得分关系
学习时长与得分存在较弱的正相关关系（r=0.047），说明适当增加学习时间有助于提高成绩，但学习效率和方法同样重要。

### 6.5 课程类别效果
图形化编程的平均得分最高，代码编程的平均代码行数最多，不同课程类型各有侧重，应根据学生特点进行合理搭配。

### 6.6 作业提交情况
大部分学生能按时提交作业（约占70%），但仍有约10%的学生存在未提交或迟交现象，需加强作业管理和督促机制。

### 6.7 出勤情况
总体出勤率约为84.1%，说明学生对编程课程的整体参与度和兴趣较高。

## 七、数据局限性

1. 本数据为 AI Mock 数据，不是真实教学场景数据，分析结论仅作为课程训练使用
2. 数据量有限，可能不能完全反映真实少儿编程教育场景的复杂性
3. Mock 数据中人为注入的缺失值和异常值模式较为简单，真实数据可能更复杂
4. 部分字段之间存在关联性可能较弱，因生成逻辑较为简化
